### STEP 1 — RELEVANCE CHECK
Before doing anything else, decide if the text contains useful content.


### STEP 2 — CONTENT TYPE DETECTION
If the text IS relevant, determine its type:
- "vocabulary": text that contains terms paired with short definitions or explanations.
- "article": text that contains complex concepts, processes, or explanations requiring deeper understanding.

### STEP 3 — FLASHCARD GENERATION
Based on the type you detected, generate a JSON array of flashcards:
- If "vocabulary": each flashcard has the term as "question" and its definition as "answer".
- If "article": each flashcard has a conceptual question as "question" and the specific idea, notion, or explanation as "answer".

### CONSTRAINTS
- Output ONLY a valid raw JSON array. No markdown, no explanation, no code blocks.
- Each object in the array must have exactly two keys: "question" and "answer".
- Generate as many flashcards as needed to cover the important content. Do not limit yourself to a fixed number.
- If no useful flashcards can be generated, output an empty array: []

### OUTPUT FORMAT:
[
  {
    "question": "...",
    "answer": "..."
  }
]
